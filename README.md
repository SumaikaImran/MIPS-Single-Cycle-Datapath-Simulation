#  MIPS Single-Cycle Datapath Simulation

A simulation of the **MIPS Single-Cycle Datapath** designed and implemented using Logisim. This project demonstrates how fundamental processor components interact in a single clock cycle to execute instructions.

## Project Overview
This project focuses on simulating a simplified version of the MIPS architecture using a **single-cycle datapath design**. Each instruction is executed in one clock cycle, showcasing the internal working of CPU components such as registers, ALU, and control unit.

## Objectives
- Understand the architecture of a MIPS processor  
- Design a single-cycle datapath  
- Simulate instruction execution in Logisim  
- Visualize data flow between components  

## Features
- Complete **single-cycle datapath implementation**
- Includes core components:
  - Program Counter (PC)
  - Instruction Memory
  - Register File
  - ALU (Arithmetic Logic Unit)
  - Data Memory
  - Control Unit
- Supports basic MIPS instructions (e.g., R-type, load/store, branch)
- Demonstrates real-time data flow and control signals
